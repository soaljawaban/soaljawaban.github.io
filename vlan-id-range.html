3. Apa yang dimaksud normal range VLAN dan extended range VLAN​


Jawaban:

Normal Range VLAN (1 – 1005)

Digunakan untuk jaringan skala kecil dan menengah.

Nomor ID 1002 s.d. 1005 dicadangkan untuk Token Ring dan FDDI VLAN.

ID 1, 1002 – 1005 secara default sudah ada dan tidak dapat dihilangkan.

Konfigurasi disimpan di dalam file database VLAN, yaitu vlan.dat. file ini disimpan dalam memori flash milkik switch.

VLAN trunking protocol (VTP), yang membantu manajemen VLAN, hanya dapat bekerja pada normal range VLAN dan menyimpannya dalam file database VLAN.

Extended Range VLANs (1006 – 4094)

Memungkinkan para service provider untuk memperluas infrastrukturnya kepada konsumen yang lebih banyak. Dibutuhkan untuk perusahaan skala besar yang membutuhkan jumlah VLAN lebih dari normal.Memiliki fitur yang lebih sedikit dibandingakn VLAN normal range.

Disimpan dalam NVRAM (file running configuration).

VTP tidak bekerja di sini.

4. apa itu vlan? dan jelaskan apa itu vlan mode trunk dan vlan mode access?


Jawaban:

VLAN (Virtual Local Area Network) adalah sekelompok perangkat pada satu LAN atau lebih yang dikonfigurasikan (menggunakan perangkat lunak pengelolaan) sehingga dapat berkomunikasi seperti halnya bila perangkat tersebut terhubung ke jalur yang sama, padahal sebenarnya perangkat tersebut berada pada sejumlah segmen LAN yang berbeda.

VLAN mode trunk adalah port yang dikonfigurasi untuk dilalui berbagai vlan.

VLAN mode access adalah port yang dikonfigurasi hanya untuk satu vlan pada switch tersebut, inti dari mode access yaitu satu port yang bisa terdaftar di satu vlan dan tidak bisa didaftarkan lebih dari satu vlan.

9. Jelaskan Terminologi VLAN Data, VLAN Default, Native VLAN, VLAN Manajemen, VLAN Voice


Jawaban:

Jelaskan Terminologi VLAN Data, VLAN Default, Native VLAN, VLAN Manajemen

Jenis VLAN berdasarkan pemberian membershipnya dibedakan menjadi beberapa jenis antara lain:

IP Subnet Address based VLAN

Authentication base VLAN

Protocol based VLAN

MAC based VLAN

Port based VLAN

Pembahasan

Virtual Local Area Network atau VLAN, merupakan suatu bentuk jaringan yang disusun teratur secara logis dalam suatu Local Area Network sehingga pengguna didalamnya dapat berinteraksi sesuai kelompok yang dibentuk. VLAN sendiri memiliki beberapa jenis menurut pemberian membershipnya sebagai berikut:

IP Subnet Address based VLAN

VLAN jenis ini memiliki kemampuan untuk bekerja pada layer dua dan tiga. Oleh karena itu, dasar dari VLAN jenis ini dapat menggunakan IP subnet address yang ada untuk pengaturan pemberian membershipnya.

Authentication based VLAN

VLAN jenis ini dapat diakses oleh perangkat secara otomatis jika berada didalamnya. Perangkat yang berada didalam VLAN ini akan dikenali setelah proses authentication terhadap user dan/atau perangkat berhasil. Protokol yang digunakan dalam proses ini adalah 802.1X.

Protocol based VLAN

Sesuai dengan nama jenis pemberian membershipnya, VLAN ini berbasis protokol jaringan. Protokol jaringan ini dapat dijadikan dasar VLAN dikarenakan pada OSI Layer dikerjakan oleh VLAN.

MAC based VLAN

VLAN jenis ini memberikan membership menurut MAC address yang dimiliki setiap perangkat didalam jaringan. MAC address yang dimiliki perangkat yang berada di dalam jaringan dan kelompoknya dapat diketahu pada tiap swicth

Port based VLAN

Pemberian membership pada jenis ini dilakukan berdasarkan konfigurasi port yang selanjutkan dimasukkan ke dalam suatu kelompok VLAN. Port hanya dapat dihubungkan dengan satu VLAN, dan jika menggunakan port trunk, maka dapat dihubungkan dengan lebih dari satu VLAN.

17. Network id, broadcast id, range ip, jumlah host dari ip 192.168.45.1/24


IP address : 192.168.45.1/24 atau

= 192.168.45.1 255.255.255.0

Cara menghitung :

Ubah IP Address dan Netmask kedalam bentuk biner, 192.168.45.1 IP Address tersebut berada di kelas C,  ambil octet ketiga dan ke empat untuk dihitung, maka :

IP address : x.x.0010 1101. 0000 0001 = 45.1

Netmask : x.x.1111 1111. 0000 0000 = 255.0

Jawab:

1. Network ID

Network ID = IP Address AND Netmask

IP address : x.x.0010 1101. 0000 0001

Netmask    : x.x.1111 1111. 0000 0000

——————————————————

AND           : x.x.0010 1101. 0000 0000 = 45.0

Sehingga Network ID nya adalah 192.168.45.0

2. Broadcast ID

Broadcast ID = IP Address OR ~Netmask

IP address : x.x.0010 1101. 0000 0001

~Netmask  : x.x.0000 0000 . 1111 1111

———————————————————

OR : x.x.0010 1101. 1111 1111 = 45.255

Sehingga Broadcast Id nya adalah 192.168.45.255

3. Range IP

Range IP = Network ID hingga(-) Broadcast ID

Sehingga Range IP nya adalah 192.168.45.0 – 192.168.45.255

4. Jumlah Host

Jumlah Host = [tex]2^{y} -2[/tex]

dimana y adalah jumlah biner 0 pada octet 3 dan 4 netmask

/24 bila di-binerkan menjadi:

1111 1111. 1111 1111. 1111 1111. 0000 0000

255         255        255            0

Sehingga Jumlah Host

= [tex]2^{8} - 2[/tex]

= [tex]256 - 2[/tex]

= [tex]254[/tex]

22. .Mendukung lalu lintas jaringan yang datang dari banyak VLAN (tagged tra c. sama baiknya dengan yang datang dari sebuah VLAN (untagged tra c). Adalah ....A. Native VLANB. VLAN DefaultC. VLAN ManajemenD. VLAN VoiceE. VLAN Data​


Mendukung lalu lintas jaringan yang datang dari banyak VLAN adalah pengertian dari A. Native VLAN. Native VLAN mendukung banyak jalur.

Penjelasan:

VLAN asli adalah VLAN yang dapat melewati port trunking tanpa memerlukan tag VLAN. Secara default, nilai VLAN asli adalah 1, yang sama dengan VLAN default. Pada dasarnya, semua paket yang melintasi port trunking memerlukan tag VLAN. Tanpa tag VLAN, paket dianggap VLAN asli dan tidak valid. Juga, VLAN asli tidak secara otomatis ditandai (atau tidak ditandai) saat diterima di port trunk. VLAN data adalah VLAN yang hanya mengatur lalu lintas di VLAN. VLAN asli adalah bentuk trunking. 37. Jelaskan Perbedaan Normal Range VLAN dan Extended Range VLAN


VLAN atau Virtual Local Area Network merupakan sebuah jaringan yang memungkinkan untuk komputer dan juga penggunanya untuk dapat berkomunikasi di dalam satu lingkungan geografis. Seolah-olah mereka ada di dalam satu LAN yang sama dan juga berbagai domain broadcaster juga multicast yang tunggal. VLAN mempunyai tujuan untuk membagi jaringan fisik menjadi beberapa domain siaran. Sehingga ini juga dapat meningkatkan kinerja pada jaringan, menghubungkan pada beberapa jenis LAN, dan juga dapat membuat keamanan jaringan.

Pembahasan :Normal Range VLAN (1 – 1005)

Digunakan pada jaringan skala kecil dan menengah. Nomor ID 1002 sampai dengan 1005 dicadangkan untuk Token Ring dan FDDI VLAN. ID 1, 1002-1005 secara default sudah ada dan juga tidak dapat dihilangkan.

Konfigurasi ini disimpan di dalam file database VLAN, yaitu vlan.dat. file ini disimpan pada memori flash milik switch.

VLAN trunking protocol atau VTP, yang dapat membantu manajemen VLAN, hanya dapat bekerja pada normal range VLAN dan juga menyimpannya di dalam file database VLAN.

Extended Range VLAN (1006 – 4094)

Memungkinkan untuk para service provider guna untuk memperluas infrastrukturnya kepada konsumen yang lebih luas dan banyak. Dibutuhkan untuk perusahaan pada skala besar yang membutuhkan jumlah VLAN lebih banyak dari normal. Memiliki fitur yang lebih sedikit dibandingkan VLAN normal range.

Disimpan di dalam NVRAM atau file running configuration. VTP tidak bekerja di sini.
